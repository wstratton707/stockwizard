## 1. 概要

US Stock Analysisは、米国株の個別銘柄について、ファンダメンタル分析（財務指標、ビジネス品質、バリュエーション）、テクニカル分析（インジケーター、チャートパターン、サポート/レジスタンス）、銘柄比較、投資レポート生成を行う対話型スキルです。

**主な特徴:**
- 4つの分析タイプ: Basic、Fundamental、Technical、Comprehensive
- WebSearchを活用したリアルタイムデータ収集
- 銘柄比較機能（AAPL vs MSFT のような2銘柄以上の比較）
- Bull/Bearケースを含む構造化レポート生成
- Buy/Hold/Sell レコメンデーション付き

**解決する問題:**
- 個別株の分析に必要な情報を手作業で集める手間を大幅に削減します
- ファンダメンタルとテクニカルを統合した一貫性のあるレポートを短時間で生成します
- バリュエーションのピア比較やフェアバリュー推定により、投資判断の材料を提供します

## 3. クイックスタート

Claudeに銘柄名を含む自然言語で指示するだけで使えます：

```
AAPLを分析して
```

Claudeが以下の流れで処理します：
1. WebSearchで最新の株価、財務指標、ニュースを収集
2. 分析タイプを判定（この例ではBasic Info）
3. 構造化されたレポートを生成

**より詳細な分析を依頼する場合：**

```
NVIDIAの包括的な投資レポートを作成してください。ファンダメンタルとテクニカルの両方を含めて、
Buy/Hold/Sellの推奨もお願いします。
```

## 4. 仕組み

### 4つの分析タイプ

Claudeはプロンプトの内容から適切な分析タイプを自動判定します：

| 分析タイプ | トリガー例 | 出力内容 |
|-----------|-----------|---------|
| **Basic** | 「AAPLの現在価格は？」「Teslaの概要」 | 株価、時価総額、主要指標、最近のニュース |
| **Fundamental** | 「NVDAの財務を分析」「Amazonは割高？」 | 財務トレンド、収益性、バランスシート、バリュエーション |
| **Technical** | 「TSLAのテクニカル分析」「AMDのトレンドは？」 | MA、RSI、MACD、サポレジ、チャートパターン |
| **Comprehensive** | 「MSFTの完全な投資レポート」 | 上記すべて + Bull/Bearケース + 推奨 |

### ワークフロー

```
プロンプト入力 → 分析タイプ判定 → WebSearchデータ収集 → 参照ファイル読み込み → 分析実行 → レポート生成
```

**Step 1: データ収集**
- WebSearchで株価、財務指標、アナリスト予想、最新ニュースを取得
- Yahoo Finance、MarketWatch、Seeking Alpha、SEC filings等の信頼性の高いソースを優先

**Step 2: 参照フレームワークの適用**
- ファンダメンタル分析: `references/fundamental-analysis.md` のビジネス品質評価、財務健全性分析、バリュエーションフレームワーク
- テクニカル分析: `references/technical-analysis.md` のインジケーター定義、チャートパターン解釈
- 財務指標: `references/financial-metrics.md` の指標定義と計算方法

**Step 3: レポート生成**
- `references/report-template.md` のテンプレート構造に従って出力
- Executive Summary → Company Overview → Investment Thesis → 分析 → Recommendation

## 5. 使用例

### 例1: クイック概要

**プロンプト:**
```
AAPLの現在の株価と主要指標を教えて
```

**Claudeの動作:**
- WebSearchで最新株価、時価総額、52週レンジ、P/E、EPSを収集
- 主要指標をテーブル形式で整理
- 直近のニュースがあれば付記

**なぜ有用か:** 銘柄の基本情報を30秒で把握できます。朝のウォッチリスト確認に最適です。

---

### 例2: ファンダメンタル分析

**プロンプト:**
```
NVIDIAの財務分析をお願いします。収益性、バランスシート、成長率を中心に。
ピア比較も含めてください。
```

**Claudeの動作:**
- 過去3-5年の売上高、利益、キャッシュフローのトレンドを収集
- ROE、ROIC、営業利益率等の収益性指標を分析
- 負債比率、流動比率でバランスシートの健全性を評価
- AMD、INTCとのピア比較テーブルを生成

**なぜ有用か:** 財務データの収集と分析を一度に完了でき、ピア比較により相対的な強弱が明確になります。

---

### 例3: テクニカル分析

**プロンプト:**
```
TSLAのテクニカル分析をしてください。トレンド、サポート/レジスタンス、
主要インジケーターの状態を教えて。
```

**Claudeの動作:**
- 現在のトレンド方向（上昇/下降/横ばい）を判定
- SMA 20/50/200の位置関係を確認
- RSI、MACD、Bollinger Bandsの状態を分析
- 主要なサポート/レジスタンスレベルを特定
- 短期・中期のテクニカルアウトルックを提示

**なぜ有用か:** エントリー/エグジットのタイミング判断に必要なテクニカル情報を体系的に整理できます。

---

### 例4: 包括的レポート（Buy/Hold/Sell推奨付き）

**プロンプト:**
```
Microsoftの完全な投資レポートを作成してください。
ファンダメンタルとテクニカルの両面から分析して、Buy/Hold/Sellの推奨をお願いします。
```

**Claudeの動作:**
- ファンダメンタル + テクニカルの両方を実行
- Bullケース（3-5つの強み）とBearケース（3-5つのリスク）を作成
- バリュエーション分析でフェアバリュー推定
- Buy/Hold/Sell の推奨 + ターゲット価格 + 確信度
- レポートを `reports/` ディレクトリに保存

**なぜ有用か:** 投資判断に必要なすべての分析が1つのレポートに集約され、買い/売りの根拠が明確になります。

---

### 例5: ピア比較（AAPL vs MSFT）

**プロンプト:**
```
AAPLとMSFTを比較分析してください。どちらが投資先として魅力的か、
ファンダメンタルとテクニカルの両面から教えて。
```

**Claudeの動作:**
- 両銘柄のデータを並行して収集
- サイドバイサイドの比較テーブル（P/E、成長率、利益率、ROE等）
- 各社の相対的な強みを定量的に比較
- テクニカル面での相対強度（Relative Strength）を分析
- どちらが魅力的か結論付け + ポートフォリオ配分提案

**なぜ有用か:** 2銘柄の横断比較を統一フォーマットで実施でき、投資先選定の判断材料になります。

---

### 例6: 決算後インパクト分析

**プロンプト:**
```
Metaの直近決算の結果と市場の反応を分析してください。
ガイダンスの変更点と今後の株価への影響もお願いします。
```

**Claudeの動作:**
- 直近の決算結果（売上、EPS）をコンセンサス予想と比較
- ガイダンスの変更点（上方修正/下方修正）を特定
- 決算発表後の株価反応（ギャップ、出来高）を分析
- 今後のカタリスト（次回決算、製品発表等）を整理

**なぜ有用か:** 決算後の投資判断に必要な情報を迅速に整理し、サプライズの程度と市場反応の妥当性を評価できます。

---

### 例7: バリュエーション評価

**プロンプト:**
```
Amazonのバリュエーションを評価してください。
P/E、PEG、EV/EBITDAでピアと比較して、割高か割安か判断してください。
```

**Claudeの動作:**
- 現在のバリュエーション指標を収集（P/E、Forward P/E、PEG、P/B、P/S、EV/EBITDA）
- 過去5年の平均バリュエーションとの比較
- ピア企業（MSFT、GOOG等）とのバリュエーション比較テーブル
- フェアバリューレンジの推定（保守的 ~ 楽観的）
- 現在価格に対するマージン・オブ・セーフティを算出

**なぜ有用か:** 複数のバリュエーション指標を統合的に評価でき、「割高なのか割安なのか」を客観的に判断する材料を提供します。

## 6. 出力の読み方

### レポートの主要セクション

| セクション | 内容 | 注目ポイント |
|-----------|------|-------------|
| Executive Summary | 投資テーゼの概要と推奨 | 最初に読むべき。レポート全体の結論 |
| Investment Thesis | Bullケース・Bearケース | リスク/リターンのバランスを確認 |
| Valuation Analysis | 指標比較 + フェアバリュー | 現在価格 vs フェアバリューの乖離に注目 |
| Technical Outlook | トレンド + キーレベル | エントリータイミングの判断材料 |
| Recommendation | Buy/Hold/Sell + ターゲット | 確信度（High/Medium/Low）に注目 |

### 推奨の読み方

**Buy推奨の場合:**
- エントリー戦略（一括、DCA、プルバック待ち）を確認
- ターゲット価格と損切りレベルをメモ
- 確信度が「High」か「Medium」かで保有サイズを調整

**Hold推奨の場合:**
- Buyへのアップグレード条件を確認
- Sellへのダウングレード条件を確認
- モニタリング優先項目を把握

**Sell推奨の場合:**
- 即座の売却か段階的な売却かを確認
- 代替投資先の提案があれば検討

### バリュエーション指標の目安

| 指標 | 割安目安 | 適正目安 | 割高目安 |
|------|---------|---------|---------|
| P/E | <15 | 15-25 | >25 |
| PEG | <1.0 | 1.0-2.0 | >2.0 |
| P/B | <1.5 | 1.5-3.0 | >3.0 |
| EV/EBITDA | <10 | 10-15 | >15 |

> これらは一般的な目安であり、セクターや成長ステージによって大きく異なります。テクノロジー成長株はP/E 30以上でも適正な場合があります。
{: .note }

## 7. Tips & ベストプラクティス

### プロンプトの書き方

- **分析タイプを明示すると精度が上がる:** 「ファンダメンタル分析をお願い」「テクニカル分析で」など
- **比較対象を指定:** 「AAPL vs MSFT vs GOOG を比較」のように複数銘柄を指定可能
- **フォーカス領域を限定:** 「バリュエーションに焦点を当てて」「キャッシュフローを重点的に」
- **出力言語の指定:** デフォルトは英語レポート。日本語レポートが必要な場合は「日本語で出力して」と指定

### 分析品質を高めるコツ

- **直近の決算前後は特にComprehensiveが有効:** 決算データが最新の状態で包括的分析ができる
- **セクター全体の状況も確認:** 個別銘柄の分析前にセクター動向を把握しておくと、業界要因と個社要因を区別しやすい
- **複数回に分けて深掘り:** まずBasicで概要を把握→気になる点をFundamentalまたはTechnicalで深掘りという段階的アプローチが効率的

## 8. 他スキルとの連携

### US Stock Analysis → Earnings Calendar

決算日を事前に把握し、決算前後のタイミングで分析を実行：

```
1. Earnings Calendar: 今週決算発表する銘柄を確認
2. US Stock Analysis: 決算前に包括的レポートを作成
3. 決算後: 結果を踏まえた再分析で推奨を更新
```

### US Stock Analysis → Technical Analyst

US Stock Analysisのテクニカルセクションで気になるパターンを発見した場合、チャート画像を使ってより詳細な分析：

```
1. US Stock Analysis: テクニカル分析でサポートライン接近を確認
2. Technical Analyst: 実際のチャート画像を使って詳細なパターン分析
3. エントリー判断: 両方の結果を統合して判断
```

### US Stock Analysis → Position Sizer

投資判断後のポジションサイズ計算：

```
1. US Stock Analysis: Buy推奨 + ターゲット価格 + 損切りレベルを取得
2. Position Sizer: エントリー価格とストップロスから適切な株数を計算
3. 実行: リスク管理されたポジションでエントリー
```

### US Stock Analysis → Value Dividend Screener

配当投資の深掘り分析：

```
1. Value Dividend Screener: 高配当バリュー株の候補リスト取得
2. US Stock Analysis: 候補銘柄を個別にComprehensive分析
3. 最終選定: 配当の持続性、成長性、バリュエーションを総合判断
```

## 9. トラブルシューティング

### データが古い

**原因:** WebSearchで取得されたデータが直近のものでない場合がある

**対処:**
- 「最新の2026年3月のデータで分析して」のように時期を明示
- 「10-Q filing 2025 Q4」のように具体的なソースを指定

### ピア比較の対象が適切でない

**原因:** Claudeが異なるセクターや規模の企業をピアとして選択する場合がある

**対処:**
- 「ピア企業はAMD、INTC、QCOMで比較して」のように明示的に指定
- 「同業種の時価総額上位5社と比較」のように条件を指定

### テクニカルデータが不足

**原因:** WebSearchでは詳細なテクニカルデータ（日足のRSI値など）が取得困難な場合がある

**対処:**
- 「TradingViewのAAPLテクニカルサマリーを確認して」とソースを指定
- Technical Analystスキルを併用し、チャート画像ベースで補完

### レポートが長すぎる

**原因:** Comprehensive分析はデフォルトで非常に詳細なレポートを生成する

**対処:**
- 「要約版で出力して」と指定
- 「Executive Summaryとバリュエーションだけ出力して」と必要なセクションを限定
